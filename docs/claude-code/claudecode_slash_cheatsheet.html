/Claude Code.

スラッシュコマンド チートシート

セッション中に / を打つとコマンド一覧が出る。定番だけ押さえれば回る。
ビルトインは60以上あって毎週増減するので、正は常に / の一覧。ここは“よく使う芯”をまとめたもの。

CONTEXT
/compact
履歴を要約して文脈を圧縮。長い作業は限界前に自分から叩く。崩れ防止の最重要コマンド。
FRESH START
/clear
会話をクリアして新規タスクへ。引きずらず作り直すのが結局速い。トークンも節約。
PROJECT
/init
CLAUDE.md を生成し、プロジェクトを把握させる。新リポで最初に1回。
1セッションの基本ループ
claude→ /init→ 作業→ /compact→ /cost→ /clear
01

セッション・コンテキスト

SESSION

文脈管理まわり。長丁場は /compact、切り替えは /clear が軸。

/clear会話をクリアして新規開始
/compact履歴を要約して圧縮（残す内容を引数指定も可）
/contextコンテキスト使用量を表示
/costトークン・コストを確認
/usage使用量・プラン上限を確認
/rewind会話／コード変更を巻き戻し（/checkpoint /undo も同じ）
/resume過去セッションを選んで再開
/export会話をエクスポート
/btw会話を汚さず横質問（履歴に残らない）
/recapここまでの要約をオンデマンド生成
02

設定・モード

CONFIG

モデル・権限・入力モードの切り替え。Vim キーバインドは /config → Editor mode で有効化（旧 /vim は v2.1.92 で削除）。

/model使用モデルを切り替え
/config設定を表示・変更（Editor mode で Vim 切替）
/permissions許可ツール・権限を設定
/effort思考の深さ（effort）を調整
/terminal-setupターミナル統合ショートカットを導入
/statuslineステータスラインを設定
03

情報・診断・アカウント

INFO

困ったらまず /help と /doctor。

/help全コマンドの一覧と説明
/status現在の状態・接続を表示
/doctorインストール・環境を診断
/release-notes更新内容を確認
/bugバグを報告
/security-review保留中の変更をセキュリティレビュー
/loginアカウントにログイン
/logoutログアウト
04

プロジェクト・拡張

EXTEND

CLAUDE.md・サブエージェント・MCP・プラグインまわり。ガバナンス設計と相性が良い。

/initCLAUDE.md を生成（プロジェクト把握）
/memoryCLAUDE.md を編集（追記は「覚えて」と頼む＝auto memory）
/add-dir作業対象ディレクトリを追加
/agentsサブエージェントを管理
/mcpMCP サーバを接続・管理
/pluginプラグイン／マーケットプレイス
/hooksフック（自動処理）を設定
05

キーボード・入力ショートカット

KEYS
キー動作
Shift + Tabモード切替（通常 → 自動承認 → Plan）
Esc生成を中断（止める）
Esc Esc入力ありなら下書きをクリア／空なら巻き戻しメニュー
Ctrl + C入力キャンセル／終了
Ctrl + Dセッション終了
Ctrl + L画面を再描画（履歴は保持）
Ctrl + Oトランスクリプトビューア（ツール実行の詳細）
Ctrl + R入力履歴を逆検索
Ctrl + B実行中の Bash／エージェントをバックグラウンド化
Ctrl + Vクリップボードの画像を貼り付け（Windows は Alt+V も）
↑ / ↓入力履歴をたどる
!Bash モード（コマンドを直接実行）
@ファイルを参照（パス補完）
/コマンドメニューを開く
\ + Enter改行（Shift+Enter／Ctrl+J も可。対応外ターミナルは /terminal-setup）
06

CLI フラグ（起動の外側）

CLI

ターミナルから claude を叩くときのオプション。

$ claude # 対話セッション開始 $ claude "バグを直して" # プロンプト付きで開始 $ claude -p "テスト実行" # 非対話（headless）で実行 $ claude -c # 直近セッションを継続 $ claude -r # セッションを選んで再開 $ claude --model opus # モデル指定 $ claude --add-dir ../shared # 追加ディレクトリを許可 $ claude -p "..." --output-format json # JSON出力 $ claude --permission-mode plan # Plan モードで起動 $ claude --effort high # 思考の深さを指定 $ claude mcp list # MCPサーバ一覧 $ claude update # 更新 $ claude doctor # 環境診断
注意：--dangerously-skip-permissions は確認を全部スキップする“yolo”モード。隔離環境以外では使わない。
07

自作コマンド & Skills

CUSTOM

よく使うプロンプトを /名前 で呼べる。チーム共有はこれが品質の効きどころ。

いまの推奨：旧 .claude/commands/ は現役だが、Skills（.claude/skills/<name>/SKILL.md）が推奨。どちらも /名前 で呼べて、Skills は Claude が自動で呼ぶこともできる。
# .claude/commands/deploy.md → /deploy で呼べる --- description: ステージングへデプロイ allowed-tools: Bash(git:*), Bash(npm:*) argument-hint: [branch] --- 現在のブランチをステージングにデプロイして。対象: $ARGUMENTS
置き場所プロジェクト直下 = 共有 ／ ~/.claude = 全プロジェクト
$ARGUMENTS本文に引数を差し込むプレースホルダ
/mcp__server__promptMCPサーバ由来のコマンド
git commit.claude/ をコミットすればチームで共有
